Gestão Acadêmica A3

Arquitetura distribuída construída para o futuro. Três microsserviços isolados, comunicação RESTful e um front-end dinâmico.

☕ Java 17 🍃 Spring Boot 3.2 ⚛️ React 18 🐬 MySQL 8 📖 Swagger

Como Iniciar o Projeto

Você pode rodar todo o ecossistema com os scripts automatizados ou subir os serviços manualmente caso prefira usar o Maven.

# Opção 1: Via Script (Recomendado no Windows)
> .\iniciar.bat

# O script subirá as 4 aplicações em terminais independentes.
# Para desligar tudo:
> .\parar.bat

⚠️ Importante: Certifique-se de que seu MySQL local está rodando na porta 3306 com credenciais root / root. O sistema cuidará da criação de todos os bancos de dados (`aluno_db`, `curso_db`, `matricula_db`).

A Arquitetura de Microsserviços

Nenhuma informação é compartilhada via banco de dados. O ecossistema é mantido em sincronia puramente via API REST HTTP usando RestTemplate, DTOs (`Records`) e Handler Globais de Exceções.

React (Vite) Front-end UI (:5173)
→
MS-Matricula API Central (:8083)
→
MS-Aluno & Curso APIs de Dados (:8081 / :8082)

Nível Profissional (Bônus)

Critérios atendidos além do MVP solicitado na disciplina:

  • Design via DTOs: Uso de Records Java 17 para isolamento de dados do Entity DB.
  • ControllerAdvice: Captura global de exceções, devolvendo sempre JSONs limpos sem expor o Tomcat.
  • Swagger OpenAPI: Especificação de endpoints nativa e injetada com anotações automáticas.
  • CORS Handling: Rotas 100% blindadas para receber chamadas de frameworks reativos SPA (React).
